# Java Basics Interview Questions and Answers

This document contains frequently asked Java Basics interview questions with concise and easy-to-understand answers for beginners, students and job seekers preparing for Java Developer interviews.

---

## 1. What is Java?

**Answer:**

Java is a high-level, object-oriented, platform-independent programming language developed by Sun Microsystems in 1995. It follows the principle **"Write Once, Run Anywhere (WORA)"**.

---

## 2. Who developed Java?

**Answer:**

Java was developed by **James Gosling** and his team at **Sun Microsystems** in 1995.

---

## 3. Why is Java called a platform-independent language?

**Answer:**

Java is platform independent because Java source code is compiled into **bytecode**, which can run on any operating system that has a **JVM (Java Virtual Machine)**.

---

## 4. What is the meaning of Write Once Run Anywhere (WORA)?

**Answer:**

A Java program is written once and can run on different operating systems without changing the code, provided a JVM is available.

---

## 5. What are the main features of Java?

**Answer:**

* Platform Independent
* Object Oriented
* Simple
* Secure
* Robust
* Portable
* Multithreaded
* Distributed
* Dynamic
* High Performance

---

## 6. What is a Java source file?

**Answer:**

A Java source file is a file that contains Java code and has the extension:

```text
.java
```

Example:

```text
HelloWorld.java
```

---

## 7. What is Bytecode?

**Answer:**

Bytecode is the intermediate code generated by the Java compiler after compiling a Java program.

It is stored in:

```text
.class
```

files.

---

## 8. What is JVM?

**Answer:**

JVM stands for **Java Virtual Machine**.

Responsibilities:

* Loading class files
* Verifying bytecode
* Executing bytecode
* Managing memory
* Garbage Collection

---

## 9. What is JRE?

**Answer:**

JRE stands for **Java Runtime Environment**.

It contains:

* JVM
* Java Libraries
* Supporting Runtime Files

JRE is used to run Java applications.

---

## 10. What is JDK?

**Answer:**

JDK stands for **Java Development Kit**.

It contains:

* JRE
* JVM
* Java Compiler (`javac`)
* Development Tools

JDK is used for developing and running Java applications.

---

## 11. Difference Between JDK, JRE and JVM

### JDK

* Used for development
* Contains JRE and development tools

### JRE

* Used for running applications
* Contains JVM and libraries

### JVM

* Executes Java bytecode

## 14. Which method is the entry point of a Java program?

**Answer:**

```java
public static void main(String[] args)
```

The JVM automatically calls this method.

---

## 15. Explain the Main Method

### public

Accessible from anywhere.

### static

Can be called without creating an object.

### void

Returns no value.

### main

Method name recognized by JVM.

### String[] args

Stores command-line arguments.

---

## 16. What is a Variable?

**Answer:**

A variable is a name given to a memory location used to store a value.

Example:

```java
int age = 25;
```

---

## 17. What is Memory Architecture of a Computer?

**Answer:**

Memory architecture consists of:

* Hard Disk for permanent storage
* RAM for temporary storage
* Processor for executing instructions

Programs are loaded from Hard Disk to RAM before execution.

---

## 18. What is Loading?

**Answer:**

The process of moving data from **Hard Disk to RAM** is called **Loading**.

---

## 19. What is Saving?

**Answer:**

The process of moving data from **RAM to Hard Disk** is called **Saving**.

---

## 20. Why is JRE Required?

**Answer:**

JRE provides the runtime environment required to execute Java programs.

Without JRE, Java applications cannot run.

---

## 21. What is the Compilation Process in Java?

**Answer:**

Step 1:

Write source code

```text
Program.java
```

Step 2:

Compile

```text
javac Program.java
```

Step 3:

Bytecode is generated

```text
Program.class
```

Step 4:

Run

```text
java Program
```

---

## 22. Which component executes Java Bytecode?

**Answer:**

**JVM (Java Virtual Machine)** executes Java bytecode.

---

## 23. Difference between Compilation and Execution

### Compilation

Converts source code into bytecode.

### Execution

Runs bytecode using JVM.

---

## 24. Can Java run without JVM?

**Answer:**

No.

JVM is mandatory for executing Java bytecode.
